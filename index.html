Frequently Asked Questions

What are the difference between var, let, and const?

var: Function-scoped, hoisted as undefined, allows redeclaration and reassignment.

let: Block-scoped, hoisted but in the Temporal Dead Zone, allows reassignment but no redeclaration.

const: Block-scoped, hoisted (Temporal Dead Zone), no redeclaration or reassignment, but object properties can change.

Best Practice: Use const by default, let when needed, and avoid var.

What are the difference between map(), forEach(), and filter()?

map(): Transforms each element and returns a new array with modified values.

forEach(): Iterates over elements but does not return anything (used for side effects).

filter(): Returns a new array with elements that pass a condition (true values).

Explain arrow functions and how they are different from regular functions
Syntax: Arrow functions are shorter and cleaner than regular functions. Regular functions require the function keyword, while arrow functions use =>. this Behavior: Arrow functions do not have their own this; they inherit it from their surrounding scope. Regular functions create their own this, which changes based on how they are called. Arguments Object: Regular functions have an arguments object, which holds all passed parameters. Arrow functions do not have arguments; they rely on rest parameters instead. Constructor Usage: Regular functions can be used as constructors with the new keyword. Arrow functions cannot be used as constructors. Best Use Cases: Arrow functions are great for short functions, especially in array methods (map, filter). Regular functions are better when this or arguments is needed, like in object methods or constructors.
How JavaScript Promises work?
A Promise in JavaScript is used to handle asynchronous operations. It represents a value that might be available now, later, or never. Promise States
A Promise has three states: Pending → Initial state, neither fulfilled nor rejected. Fulfilled → The operation was successful. Rejected → The operation failed.
A Promise starts in a pending state. It either resolves (fulfilled) or rejects (failed). You handle the result using .then() for success and .catch() for errors.
Key Methods .then(callback) → Runs when the Promise is fulfilled. .catch(callback) → Runs when the Promise is rejected. .finally(callback) → Runs regardless of success or failure.

How Closures Work in JavaScript

A closure is a function that remembers the variables from its outer scope, even after the outer function has finished executing.

How It Works:

When a function is defined inside another function, it captures the outer function’s variables.

Even if the outer function returns, the inner function still has access to those variables.

Why Closures Are Useful?

Data Privacy: You can create private variables that are not accessible from outside.

State Persistence: Values remain in memory between function calls.

Function Factories: Generate customized functions dynamically.

Callbacks & Event Handlers: Used in asynchronous programming.
